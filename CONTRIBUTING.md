# Contributing to Slate

Thank you for your interest in contributing to Slate! We appreciate your help in making this project better for everyone.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue on GitHub. Include:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Your browser and OS versions.
- Screenshots if applicable.

### Suggesting Features
Have an idea for a new feature? We'd love to hear it! Open an issue and describe:
- The problem it solves.
- Your proposed solution.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Make your changes, ensuring they follow the project's coding standards.
4. Run `npm run lint` to check for style issues.
5. Submit a pull request with a clear description of what you've done.

## Development Setup

1. Fork and clone the repo.
2. Install dependencies: `npm install`.
3. Start the dev server: `npm run dev`.

## Coding Standards

- Use **TypeScript** for all new code.
- Follow the existing formatting and style (ESLint is configured to help).
- Keep components focused and modular.

## Code of Conduct

Please be respectful and welcoming to all contributors. We aim to foster a collaborative and inclusive environment.
